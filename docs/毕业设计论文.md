# 基于 FastAPI + Vue3 的 Linux 运行监控系统的设计与实现

## 毕业设计（论文）文档

---

## 摘要

随着云计算和容器化技术的广泛应用，企业的服务器大多运行在 Linux 平台之上，及时掌握系统的资源使用状况、发现潜在瓶颈并采取预防措施，成为保障业务稳定运行的重要前提。本文设计并实现了一套基于 Python 后端框架 FastAPI 和前端框架 Vue3 的轻量级 Linux 运行监控系统。该系统能够实时采集 CPU、内存、磁盘 I/O、网络带宽等系统指标，通过 WebSocket 实现数据的实时推送，并使用 ECharts 进行可视化展示。系统采用 SQLite3 作为后端存储，支持历史数据查询和数据导出功能，同时实现了告警机制和进程监控功能。经过测试，系统运行稳定，各项功能满足设计要求。

**关键词**：Linux 监控；FastAPI；Vue3；psutil；ECharts

## 第一章 绪论

### 1.1 研究背景与意义

在云计算和容器化技术广泛应用的今天，企业的服务器大多运行在 Linux 平台之上。Linux 系统以其稳定、开源、安全的特性，成为服务器领域的首选操作系统。然而，如何及时掌握系统的资源使用状况、发现潜在瓶颈并采取预防措施，是保障业务稳定运行的重要前提。

传统的 Linux 系统监控工具如 top、htop、vmstat 等可以查看 CPU 负载、内存使用和 I/O 状况，但这些工具往往需要人工交互，缺乏可视化和历史分析能力。Nagios、Zabbix 等企业级监控套件功能强大，但部署复杂，不适合前后端分离的 Web 应用集成。

因此，构建一套基于 Python 和现代前端框架的轻量级监控系统，既能及时采集系统指标，又能通过网页动态展示，是面向教学与工程实践的有益探索。

### 1.2 国内外研究现状

Linux 系统监控一直是运维领域的重要课题。常用的监控方案包括：

1. **命令行工具**：top、htop、vmstat、iostat 等，提供基础的系统指标查看功能
2. **SNMP 监控**：通过 SNMP 协议采集设备指标，支持大规模网络设备监控
3. **Zabbix**：企业级开源监控解决方案，功能完善但部署复杂
4. **Prometheus + Grafana**：云原生监控架构，适合容器化环境

上述方案各有优缺点，本文旨在设计一套轻量级、易部署的监控系统，适合教学环境和小型业务场景。

### 1.3 研究目标与内容

本文的主要研究目标是设计并实现一套基于 FastAPI + Vue3 的 Linux 运行监控系统，具体包括：

1. 监控数据采集模块：使用 psutil 库采集 CPU、内存、磁盘、网络指标
2. 数据存储模块：使用 SQLite3 存储历史数据
3. 后端服务模块：基于 FastAPI 构建 REST API 和 WebSocket 服务
4. 前端展示模块：使用 Vue3 和 ECharts 实现可视化仪表盘
5. 扩展功能：告警机制、进程监控、数据导出

## 第二章 相关技术概述

### 2.1 Linux /proc 文件系统

Linux 内核通过 /proc 虚拟文件系统向用户空间程序提供系统状态信息。/proc 目录下的文件不是真实的文件，而是内核数据的接口，读取这些文件即可获取系统实时状态。

常用的监控相关文件包括：
- /proc/cpuinfo：CPU 型号和核心数信息
- /proc/meminfo：内存使用统计
- /proc/stat：各类 CPU 时间统计
- /proc/diskstats：磁盘 I/O 统计
- /proc/net/dev：网络设备统计

### 2.2 psutil 库

psutil 是一个跨平台的 Python 库，用于获取系统和进程信息。它封装了对 /proc 文件系统的读取，提供统一的 Python API。

主要函数：
- cpu_percent()：获取 CPU 使用率
- virtual_memory()：获取内存使用情况
- disk_io_counters()：获取磁盘 I/O 统计
- net_io_counters()：获取网络 I/O 统计

### 2.3 FastAPI 框架

FastAPI 是近年兴起的现代 Python Web 框架，基于标准的 Python 类型提示构建，具有高性能、易于开发的特点。

主要特性：
- 高性能：与 Node.js 和 Go 不相上下
- 自动生成 API 文档
- 支持异步编程
- 原生支持 WebSocket

### 2.4 Vue3 框架

Vue3 是渐进式 JavaScript 框架，采用 Composition API 组织代码，配合 ECharts 可快速构建交互式仪表盘。

主要特性：
- 响应式数据绑定
- 组件化开发
- Composition API
- 虚拟 DOM

### 2.5 ECharts 可视化库

ECharts 是开源的 JavaScript 可视化库，支持折线图、柱状图、热力图、仪表盘等多种图表类型，提供增量渲染和 WebSocket 流式数据支持。

#### 3.1.4 前端展示

- 历史数据查询：支持时间范围选择

## 第五章 关键技术实现

### 5.1 监控数据采集

#### 5.1.1 CPU 采集

```python
def get_cpu():
    per_core = psutil.cpu_percent(interval=0.1, percpu=True)
    total = psutil.cpu_percent(interval=0.1)
    return {
        "percent": round(total, 2),
        "per_core": [{"core": i, "percent": round(p, 2)} for i, p in enumerate(per_core)],
        "count": psutil.cpu_count()
    }
```

注意：psutil.cpu_percent() 首次调用返回 0，需要预热处理。

#### 5.1.2 内存采集

```python
def get_memory():
    mem = psutil.virtual_memory()
    return {
        "total": round(mem.total / (1024**3), 2),
        "available": round(mem.available / (1024**3), 2),
        "used": round(mem.used / (1024**3), 2),
        "percent": round(mem.percent, 2)
    }
```

#### 5.1.3 进程监控

```python
def get_processes(top_n=10):
    processes = []
    for proc in psutil.process_iter(['pid', 'name', 'cpu_percent', 'memory_percent', 'status']):
        try:
            info = proc.info
            if info['cpu_percent'] is None:
                info['cpu_percent'] = 0
            if info['memory_percent'] is None:
                info['memory_percent'] = 0
            processes.append(info)
        except (psutil.NoSuchProcess, psutil.AccessDenied):
            pass
    processes.sort(key=lambda x: x['cpu_percent'], reverse=True)
    return processes[:top_n]
```

    alerts = []

            continue

## 第八章 总结与展望

### 8.1 总结

本文设计并实现了一套基于 FastAPI + Vue3 的 Linux 运行监控系统，主要完成了以下工作：

1. 实现了 CPU、内存、磁盘 I/O、网络带宽的实时监控采集
2. 使用 SQLite3 实现了历史数据的存储和查询

5. 实现了告警机制、进程监控和数据导出功能

系统经过测试，各项功能满足设计要求，运行稳定。

### 8.3 展望

1. 容器化监控：支持 Docker、Kubernetes 监控
