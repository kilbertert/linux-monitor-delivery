# 快速开始指南

## 环境准备

### 后端环境

```bash
# 进入后端目录
cd backend

# 创建虚拟环境
python3 -m venv .venv

# 激活虚拟环境
# Linux/Mac:
source .venv/bin/activate
# Windows:
.venv\Scripts\activate

# 安装依赖
pip install -r requirements.txt

# 启动后端服务
uvicorn main:app --host 0.0.0.0 --port 8000
```

### 前端环境

```bash
# 进入前端目录
cd frontend

# 安装依赖
npm install

# 启动开发服务器
npm run dev

# 或构建生产版本
npm run build
```

## 访问地址

- 后端 API：http://localhost:8000
- API 文档：http://localhost:8000/docs
- 前端页面：http://localhost:5173

## 功能页面

| 页面 | 路径 | 说明 |
|------|------|------|
| 仪表盘 | / | 实时监控数据展示 |
| 历史数据 | /history | 历史指标查询 |
| 进程监控 | /processes | 进程列表 |
| 告警管理 | /alerts | 告警规则配置 |
| 数据导出 | /export | CSV 数据导出 |

## API 接口示例

### 获取当前指标

```bash
curl http://localhost:8000/metrics/current
```

### 获取历史数据

```bash
curl "http://localhost:8000/metrics/history?metric_type=cpu&start_time=$(date -d '1 hour ago' +%s)000&end_time=$(date +%s)000"
```

### 获取进程列表

```bash
curl "http://localhost:8000/processes?top=10&sort=cpu"
```

## WebSocket 连接

```javascript
const ws = new WebSocket('ws://localhost:8000/ws/metrics');

ws.onopen = () => {
  console.log('Connected');
  ws.send(JSON.stringify({ action: 'subscribe', metrics: ['cpu', 'memory'] }));
};

ws.onmessage = (event) => {
  const data = JSON.parse(event.data);
  console.log(data);
};
```

## 数据导出

访问 http://localhost:5173/export 页面，选择指标类型和时间范围，点击"导出 CSV"按钮下载数据。

## 常见问题

### 1. 前端无法连接后端

检查后，确保端服务是否启动 IP 地址配置正确（Windows 和 WSL 之间的网络配置）。

### 2. 数据未更新

刷新页面，检查 WebSocket 连接状态。

### 3. 进程监控无数据

部分进程可能因为权限问题无法获取，确保以管理员权限运行。
